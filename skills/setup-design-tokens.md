# setup-design-tokens.md — Design Token Layer Setup

> **Loaded by:** `frontend-architect` on new projects to establish the token layer before
> producing component specs.
>
> **When to use:** Load this skill when a new frontend project is starting and no token layer
> exists yet. For existing projects with an established `tailwind.config`, skip this skill
> and read the existing config to confirm which tokens are in use.

---

## What This Skill Produces

A `tailwind.config.js` (or `tailwind.config.ts`) that maps all `design_style.md` functional
color roles to named theme extension tokens. This file becomes the single source of truth
for all color, spacing, and typography values in the project. No component should reference
a raw Tailwind hue (e.g. `blue-600`) directly — it should reference a semantic token
(e.g. `accent`) that maps to the hue.

---

## Steps

### 1. Confirm the accent hue with the human

Before writing the config, ask: "Which accent hue should this project use?" The default is
Blue. Do not assume and proceed.

### 2. Map color roles to tokens

Using the Functional Mapping Matrix from `resources/design_style.md`, define these semantic
tokens in the Tailwind theme extension:

| Token name | Light mode | Dark mode | Source rule |
|---|---|---|---|
| `colors.background` | `slate-50` | `zinc-950` | App Background |
| `colors.surface` | `white` | `slate-900` | Surface / Card |
| `colors.border` | `slate-200` | `slate-800` | Muted Boundary |
| `colors.accent` | `[hue]-600` | `[hue]-500` | Primary Accent |
| `colors.accent-subtle` | `[hue]-50` | `[hue]-950` | Accent Subtle |
| `colors.text-intense` | `slate-900` | `slate-50` | Text Intense |
| `colors.text-default` | `slate-700` | `slate-300` | Body text |
| `colors.text-muted` | `slate-400` | `slate-500` | Secondary labels |

Replace `[hue]` with the confirmed accent hue name (e.g. `blue`, `orange`, `emerald`).

### 3. Define semantic state colors

These are fixed regardless of accent hue:

| Token name | Value | Usage |
|---|---|---|
| `colors.success` | `emerald-600` / `emerald-500` | Positive status, confirmation |
| `colors.warning` | `amber-500` / `amber-400` | Cautionary state |
| `colors.danger` | `red-600` / `red-500` | Destructive action, error state |

### 4. Define typography tokens

```js
fontFamily: {
  interface: ['Inter', 'SF Pro Display', 'Geist Sans', 'sans-serif'],
  data: ['JetBrains Mono', 'SF Mono', 'Geist Mono', 'monospace'],
}
```

All interface copy uses `font-interface`. All technical metadata, metrics, table IDs, and
code strings use `font-data`.

### 5. Confirm spacing scale

Use Tailwind's default 4px base scale. Do not add custom arbitrary spacing values. If a
spacing need cannot be met by the default scale, flag it rather than reaching for `[]`.

### 6. Write the config

```js
// tailwind.config.js — generated by setup-design-tokens
module.exports = {
  content: ['./src/**/*.{ts,tsx}'],
  theme: {
    extend: {
      colors: {
        background: { DEFAULT: '#f8fafc', dark: '#09090b' },   // slate-50 / zinc-950
        surface:    { DEFAULT: '#ffffff', dark: '#0f172a' },   // white / slate-900
        border:     { DEFAULT: '#e2e8f0', dark: '#1e293b' },   // slate-200 / slate-800
        accent:     { DEFAULT: '<hue>-600', dark: '<hue>-500' },
        'accent-subtle': { DEFAULT: '<hue>-50', dark: '<hue>-950' },
        'text-intense':  { DEFAULT: '#0f172a', dark: '#f8fafc' },
        'text-default':  { DEFAULT: '#334155', dark: '#cbd5e1' },
        'text-muted':    { DEFAULT: '#94a3b8', dark: '#64748b' },
        success: { DEFAULT: '#059669', dark: '#10b981' },
        warning: { DEFAULT: '#f59e0b', dark: '#fbbf24' },
        danger:  { DEFAULT: '#dc2626', dark: '#ef4444' },
      },
      fontFamily: {
        interface: ['Inter', 'SF Pro Display', 'Geist Sans', 'sans-serif'],
        data: ['JetBrains Mono', 'SF Mono', 'Geist Mono', 'monospace'],
      },
    },
  },
  plugins: [],
}
```

---

## Reviewer Check

Before `frontend-architect` proceeds to component specs, verify:

- [ ] Accent hue confirmed with the human
- [ ] No raw Tailwind hue values in the config (e.g. `blue-600`) — all replaced with
      semantic `<hue>-weight` substitutions against the confirmed hue
- [ ] `font-data` is configured for monospace use
- [ ] Config file committed or staged before any component code is written
