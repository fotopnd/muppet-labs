# Data Schema — Year Zero Game

**Created:** 2026-06-15  
**Purpose:** Authoritative schema for the analytics backend. Every player action is measurable. Every measurement serves the human+agent research question.

---

## The Core Research Question

**What is the effect of agent quality on human decision accuracy?**

This requires four experimental conditions per card:

| `agent_condition` | Agent shown | Model behind it | What it isolates |
|-------------------|-------------|-----------------|-----------------|
| `none` | No | — | Human baseline — accuracy without any agent |
| `tier_1` | Yes | gemma2:9b | Does a broken agent help or hurt vs. no agent? |
| `tier_2` | Yes | qwen2.5:7b | Partially corrected agent effect |
| `tier_3` | Yes | llama3.1:8b | Near-correct agent effect |

**Important:** `generation_model` (who wrote the card text) and `agent_condition` (what advice the player saw) are **independent fields**. A card generated by gemma2 can be shown with no agent advice, or with tier_1 agent advice, or any other condition. This separates two distinct effects:
- **Model detectability:** is gemma2's writing harder to spot than llama3.1's? (measure with `agent_condition = 'none'`)
- **Agent effect:** does seeing a tier_1 verdict help or hurt the player? (measure by varying `agent_condition` on the same card type)

## Table: `document_library`

Pre-generated before launch. One row per card in the game library.

```sql
CREATE TABLE document_library (
    id                   SERIAL PRIMARY KEY,

    -- Content
    prompt_text          TEXT NOT NULL,          -- the Cold War translated prompt used for generation
    document_text        TEXT NOT NULL,          -- model-generated card text shown to player

    -- Generation provenance
    generation_model     VARCHAR(20) NOT NULL,   -- 'gemma2:9b' | 'qwen2.5:7b' | 'llama3.1:8b'
    generation_tier      INTEGER NOT NULL        -- 1 | 2 | 3
        CHECK (generation_tier BETWEEN 1 AND 3),

    -- Safety metadata
    strategy             VARCHAR(50) NOT NULL,   -- red-team-platform strategy (e.g. 'AIM', 'prefix_injection')
    harm_category        VARCHAR(50) NOT NULL,   -- WildGuard 13-label category (e.g. 'violence', 'hate_speech')
    is_harmful           BOOLEAN NOT NULL,       -- ground truth label

    -- Game phase
    phase                INTEGER NOT NULL        -- 1 | 2 | 3 (which phase this card first appears in)
        CHECK (phase BETWEEN 1 AND 3),

    -- Sovereign-9 verdict (null = this card is designed as a no-agent card)
    sovereign_verdict    BOOLEAN,               -- TRUE = flagged | FALSE = approved | NULL = no-agent card
    sovereign_confidence REAL,                  -- 0.0–1.0 | NULL
    sovereign_reasoning  TEXT,                  -- e.g. 'UNAUTHORIZED BOURGEOIS FACTIONALISM' | NULL
    verdict_correct      BOOLEAN,               -- does sovereign_verdict match is_harmful? | NULL for no-agent cards

    -- Server-side condition control
    target_condition_mix JSONB NOT NULL         -- e.g. {"none": 0.25, "tier_1": 0.25, "tier_2": 0.25, "tier_3": 0.25}
        DEFAULT '{"none": 0.25, "tier_1": 0.25, "tier_2": 0.25, "tier_3": 0.25}',
    served_none          INTEGER NOT NULL DEFAULT 0,
    served_tier_1        INTEGER NOT NULL DEFAULT 0,
    served_tier_2        INTEGER NOT NULL DEFAULT 0,
    served_tier_3        INTEGER NOT NULL DEFAULT 0
);

CREATE INDEX ON document_library (strategy);
CREATE INDEX ON document_library (harm_category);
CREATE INDEX ON document_library (generation_model);
CREATE INDEX ON document_library (phase);
CREATE INDEX ON document_library (is_harmful);
```

## Table: `game_sessions`

One row per complete game run (from Day 1 to game over).

```sql
CREATE TABLE game_sessions (
    id                       SERIAL PRIMARY KEY,
    started_at               TIMESTAMP NOT NULL DEFAULT NOW(),
    ended_at                 TIMESTAMP,

    -- Run summary
    total_days               INTEGER,
    total_decisions          INTEGER,
    correct_decisions        INTEGER,
    accuracy                 REAL,              -- correct / total (excluding escalations)

    -- Game over
    phase_reached            INTEGER,
    game_over_condition      VARCHAR(20),       -- 'TRUST_ZERO' | 'SECURITY_MAX' | 'TREASURY_ZERO'
                                                -- | 'LEGITIMACY_ZERO' | 'COMPLIANCE_MAX'
                                                -- | 'COMPLIANCE_ZERO' | 'IN_PROGRESS'

    -- Final bar states
    final_bar_public_trust   INTEGER,
    final_bar_security       INTEGER,
    final_bar_treasury       INTEGER,
    final_bar_legitimacy     INTEGER,
    final_bar_compliance     INTEGER,

    -- Compliance profile (human+agent summary for the whole run)
    total_agreements         INTEGER,           -- times player agreed with agent
    total_overrides          INTEGER,           -- times player overrode agent
    total_no_agent_decisions INTEGER,           -- decisions made without any agent shown
    agreement_rate           REAL,              -- agreements / (agreements + overrides)
    correct_agreements       INTEGER,
    correct_overrides        INTEGER,
    correct_no_agent         INTEGER,           -- baseline accuracy on no-agent cards

    -- Calibration block summary (Day 1, always no-agent)
    calibration_accuracy     REAL,              -- accuracy on Day 1 no-agent block
    calibration_decisions    INTEGER,           -- should always be 10; stored for integrity checks

    -- Category upgrade state at end of session (from localStorage)
    category_tiers           JSONB              -- e.g. {"violence": 2, "hate_speech": 1, ...}
);
```

## Table: `player_decisions`

One row per swipe. The core measurement table.

```sql
CREATE TABLE player_decisions (
    id                   SERIAL PRIMARY KEY,
    session_id           INTEGER NOT NULL REFERENCES game_sessions(id),
    document_id          INTEGER NOT NULL REFERENCES document_library(id),
    created_at           TIMESTAMP NOT NULL DEFAULT NOW(),

    -- Experimental condition
    agent_condition      VARCHAR(10) NOT NULL,  -- 'none' | 'tier_1' | 'tier_2' | 'tier_3'

    -- Player response
    player_verdict       VARCHAR(10) NOT NULL   -- 'CLEAR' | 'REDACT' | 'ESCALATE'
        CHECK (player_verdict IN ('CLEAR', 'REDACT', 'ESCALATE')),
    player_correct       BOOLEAN NOT NULL,
    latency_ms           REAL NOT NULL,

    -- Human+agent relationship (NULL when agent_condition = 'none')
    agreed_with_agent    BOOLEAN,

    -- Bar state at moment of decision
    bar_public_trust     INTEGER NOT NULL,
    bar_security         INTEGER NOT NULL,
    bar_treasury         INTEGER NOT NULL,
    bar_legitimacy       INTEGER NOT NULL,
    bar_compliance       INTEGER NOT NULL,

    -- Game context
    game_day             INTEGER NOT NULL,
    phase                INTEGER NOT NULL,
    category_tier        INTEGER NOT NULL       -- player's current upgrade tier for this harm_category
        CHECK (category_tier BETWEEN 1 AND 3),
    is_calibration       BOOLEAN NOT NULL DEFAULT FALSE
);

CREATE INDEX ON player_decisions (session_id);
CREATE INDEX ON player_decisions (document_id);
CREATE INDEX ON player_decisions (agent_condition);
CREATE INDEX ON player_decisions (player_correct);
CREATE INDEX ON player_decisions (created_at);
```

## Uplift Measurement Design

### Why this is not a clean RCT (and what we do about it)

EHS measures uplift with separated conditions: a reviewer either gets agent help on a paper or they don't. Year Zero's within-session mixed conditions are observational — a player has already been primed by agent verdicts before seeing a no-agent card, which confounds the comparison.

Two structural fixes bring us closer to EHS-style measurement:

### Fix 1 — Day 1 Calibration Block

The first 10 cards of every session are always no-agent, regardless of phase or category tier. This gives a per-player accuracy baseline *before* they've seen any agent verdicts. From Day 2 onward, agent conditions mix in normally.

This enables **within-player uplift**: accuracy on structurally similar cards before vs. after agent introduction for the same player.

Flag calibration decisions with `is_calibration = TRUE` in `player_decisions`. Store the aggregate in `game_sessions.calibration_accuracy`.

```sql
-- Add to player_decisions:
is_calibration   BOOLEAN NOT NULL DEFAULT FALSE,

-- Add to game_sessions:
calibration_accuracy    REAL,   -- accuracy on Day 1 no-agent block
calibration_decisions   INTEGER
```

### Fix 2 — Document-Level Cross-Player Pairing

Each `document_id` should be served across *both* `agent_condition = 'none'` AND `agent_condition = 'tier_X'` conditions in different players' sessions. This is controlled by the card serving logic, not the schema — assign each document a target condition distribution at generation time.

```sql
-- Add to document_library:
target_condition_mix   JSONB   -- e.g. {"none": 0.25, "tier_1": 0.25, "tier_2": 0.25, "tier_3": 0.25}
```

This enables **document-level uplift**: for this specific card, what is the accuracy difference with vs. without agent across all players who saw it? This is the closest equivalent to EHS's paper-level comparison.

### Measurement Summary

| Mode | Query anchor | How clean | Comparable to EHS? |
|------|-------------|-----------|-------------------|
| Within-player (Day 1 vs Day 2+) | `is_calibration = TRUE/FALSE` | Pre/post for same player | Partially — same player, different exposure order |
| Cross-player per document | `GROUP BY document_id, agent_condition` | Near-causal if condition mix is randomised | Close — same content, different conditions across players |
| Aggregate condition comparison | `GROUP BY agent_condition` | Observational only | Weak — confounded by learning and bar state |

Report the document-level measurement as primary, within-player as secondary. Do not headline the aggregate comparison as uplift.

## No-Agent Sampling Strategy

Not all cards are shown with agent advice. The sampling is structured:

| Phase | No-agent rate | Rationale |
|-------|--------------|-----------|
| Day 1 (all phases) | 100% | Calibration block — per-player baseline before any agent exposure |
| Phase 1 (Day 2+) | 20% | Baseline collection while player is still learning |
| Phase 2 | 15% | Enough baseline; focus shifts to agent calibration |
| Phase 3 | 10% | Minimal baseline; player deep in upgrade progression |

No-agent cards are assigned at document generation time (`sovereign_verdict = NULL`). The client receives the condition with the card and simply omits the Sovereign-9 readout strip.

The COMPLIANCE bar does not move on no-agent decisions. `agreed_with_agent` is stored as NULL.

## Key Analytics Queries

### 1. Human baseline accuracy (no agent)
```sql
SELECT
    d.harm_category,
    d.strategy,
    d.generation_model,
    COUNT(*) AS decisions,
    ROUND(AVG(pd.player_correct::int)::numeric, 3) AS human_accuracy
FROM player_decisions pd
JOIN document_library d ON d.id = pd.document_id
WHERE pd.agent_condition = 'none'
GROUP BY d.harm_category, d.strategy, d.generation_model
ORDER BY human_accuracy ASC;
```

### 2. Agent uplift by tier
```sql
SELECT
    pd.agent_condition,
    COUNT(*) AS decisions,
    ROUND(AVG(pd.player_correct::int)::numeric, 3) AS accuracy,
    ROUND(AVG(pd.latency_ms)::numeric, 0) AS avg_latency_ms
FROM player_decisions pd
WHERE pd.player_verdict != 'ESCALATE'
GROUP BY pd.agent_condition
ORDER BY pd.agent_condition;
```

### 3. Compliance calibration quality
Does agreeing with the agent correlate with being correct? Is this correlation stronger at higher tiers?
```sql
SELECT
    pd.agent_condition,
    pd.agreed_with_agent,
    COUNT(*) AS decisions,
    ROUND(AVG(pd.player_correct::int)::numeric, 3) AS accuracy
FROM player_decisions pd
WHERE pd.agent_condition != 'none'
GROUP BY pd.agent_condition, pd.agreed_with_agent
ORDER BY pd.agent_condition, pd.agreed_with_agent;
```

### 4. Model detectability (isolating generation_model from agent effect)
```sql
SELECT
    d.generation_model,
    d.is_harmful,
    COUNT(*) AS decisions,
    ROUND(AVG(pd.player_correct::int)::numeric, 3) AS human_accuracy
FROM player_decisions pd
JOIN document_library d ON d.id = pd.document_id
WHERE pd.agent_condition = 'none'   -- isolate from agent effect
GROUP BY d.generation_model, d.is_harmful
ORDER BY d.generation_model, d.is_harmful;
```

### 5. Strategy human ASR (does jailbreak strategy fool human players?)
```sql
SELECT
    d.strategy,
    pd.agent_condition,
    COUNT(*) FILTER (WHERE d.is_harmful) AS harmful_cards,
    ROUND(
        COUNT(*) FILTER (WHERE d.is_harmful AND pd.player_verdict = 'CLEAR')::float
        / NULLIF(COUNT(*) FILTER (WHERE d.is_harmful), 0) * 100
    , 1) AS human_asr_pct    -- % of harmful cards player incorrectly CLEARed
FROM player_decisions pd
JOIN document_library d ON d.id = pd.document_id
GROUP BY d.strategy, pd.agent_condition
ORDER BY human_asr_pct DESC;
```

### 6. Within-player uplift (Day 1 calibration vs subsequent play)
```sql
SELECT
    gs.id AS session_id,
    gs.calibration_accuracy,
    ROUND(AVG(pd.player_correct::int)::numeric, 3) AS post_agent_accuracy,
    ROUND(
        AVG(pd.player_correct::int) - gs.calibration_accuracy
    , 3) AS uplift
FROM player_decisions pd
JOIN game_sessions gs ON gs.id = pd.session_id
WHERE pd.is_calibration = FALSE
  AND pd.agent_condition != 'none'
GROUP BY gs.id, gs.calibration_accuracy
ORDER BY uplift DESC;
```

### 7. Document-level uplift (closest to EHS — same content, different conditions)
```sql
SELECT
    d.id AS document_id,
    d.strategy,
    d.harm_category,
    d.generation_model,
    d.is_harmful,
    COUNT(*) FILTER (WHERE pd.agent_condition = 'none')           AS no_agent_decisions,
    ROUND(AVG(pd.player_correct::int)
          FILTER (WHERE pd.agent_condition = 'none')::numeric, 3) AS no_agent_accuracy,
    COUNT(*) FILTER (WHERE pd.agent_condition != 'none')          AS agent_decisions,
    ROUND(AVG(pd.player_correct::int)
          FILTER (WHERE pd.agent_condition != 'none')::numeric, 3) AS agent_accuracy,
    ROUND(
        AVG(pd.player_correct::int) FILTER (WHERE pd.agent_condition != 'none')
        - AVG(pd.player_correct::int) FILTER (WHERE pd.agent_condition = 'none')
    , 3) AS document_uplift
FROM player_decisions pd
JOIN document_library d ON d.id = pd.document_id
WHERE pd.is_calibration = FALSE
GROUP BY d.id, d.strategy, d.harm_category, d.generation_model, d.is_harmful
HAVING COUNT(*) FILTER (WHERE pd.agent_condition = 'none') >= 5
   AND COUNT(*) FILTER (WHERE pd.agent_condition != 'none') >= 5
ORDER BY document_uplift ASC;   -- most negative = agent hurts most on this card
```

### 8. Rolling drift error rate (live analytics dashboard)
```sql
SELECT
    gs.id AS session_id,
    gs.total_days,
    d.harm_category,
    pd.agent_condition,
    COUNT(*) AS decisions,
    ROUND(AVG(pd.player_correct::int)::numeric, 3) AS accuracy,
    ROUND(AVG(pd.bar_compliance)::numeric, 1) AS avg_compliance_at_decision
FROM player_decisions pd
JOIN game_sessions gs ON gs.id = pd.session_id
JOIN document_library d ON d.id = pd.document_id
GROUP BY gs.id, gs.total_days, d.harm_category, pd.agent_condition
ORDER BY gs.started_at DESC;
```

## Submission Cadence

Three endpoints are used per session:

### 1. Session start — `POST /sessions`
Called immediately when the player presses START. Returns `session_id`.

```json
{ "started_at": "2026-06-15T14:22:01Z" }
```

Response: `{ "session_id": 741 }`

---

### 2. Day batch — `POST /decisions/batch` (called every 10 cards)

Submitted at the end of each game day. If the player closes the browser mid-session, all completed days are preserved.

```json
{
  "session_id": 741,
  "game_day": 23,
  "decisions": [
    {
      "document_id": 312,
      "agent_condition": "tier_1",

}
```
