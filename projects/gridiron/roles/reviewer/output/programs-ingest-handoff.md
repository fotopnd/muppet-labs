## Deviations from Brief

1. **stadium_cap populated from identity table** — The brief's Brand table has no Cap column. The identity table has a `Cap` (real stadium capacity) column. The seed script stores this real cap value in `stadium_cap`. The brief note says "Fictional stadium = real cap ±10% random" — applying that randomization is out of scope for this brief (no such instruction in the seed script spec). Reviewer should confirm whether storing raw real cap or applying ±10% is desired for this seeding step.

2. **primary_color == secondary_color in source data** — The UNIVERSE-SEEDING.md Brand tables have identical hex values in the Primary and Secondary columns for all programs (e.g., Ohio Polytechnic: `#C8A027` for both). This appears intentional (secondary color refinement deferred). The seeder stores what the source doc provides. No validation check for primary != secondary was added since the brief doesn't require it.

3. **`conglomerate_id` is nullable in models.py** — The SQLAlchemy model marks this as `nullable=True` to match the brief's schema spec (`INTEGER REFERENCES conglomerates(id)` with no NOT NULL). All seeded programs have non-NULL conglomerate_id in practice. The brief's Section 3.2 full schema does say `NOT NULL` on this field — if the reviewer wants stricter, the model and migration can be tightened.

4. **Migration deliverable location** — The brief mentions `db/migrations/001_programs_schema.sql` as a deliverable, but the project uses Alembic (not raw SQL files). The migration lives in `alembic/versions/33f31770f03e_programs_schema.py` and was applied via `alembic upgrade head`. The `db/migrations/` directory does not exist. If a raw SQL file is also needed, it can be generated.
